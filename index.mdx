## Overview

The Bytedesk API provides a comprehensive set of endpoints for managing customer support, messaging, and communication workflows. Our RESTful API is designed to be developer-friendly with consistent response formats and detailed error messages.

## Authentication

All API requests require authentication using Bearer tokens. Include your API key in the Authorization header:

```bash
Authorization: Bearer <token>
```

## Base URL

All API requests should be made to:

```
http://api.weiyuai.cn
```

## Response Format

All API responses follow a consistent JSON format:

```json
{
  "success": true,
  "data": {},
  "message": "Operation completed successfully"
}
```

## Error Handling

When an error occurs, the API returns an appropriate HTTP status code along with an error response:

```json
{
  "error": 400,
  "message": "Bad Request - Invalid parameters"
}
```
