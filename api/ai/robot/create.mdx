## 概述

创建一个新的 AI 智能客服机器人。

<div style={{ marginTop: 24, marginBottom: 24 }}>
  <ApiPlayground
    method="POST"
    endpoint="/api/v1/robot"
  />
</div>

## 接口说明

此接口用于创建 AI 智能客服机器人。可以配置机器人的名称、头像、使用的 LLM 模型、提示词等参数。

## 请求头

```
Authorization: Bearer <access_token>
Content-Type: application/json
```

## 请求体

```json
{
  "name": "智能客服小助手",
  "avatar": "https://example.com/avatar.png",
  "description": "基于大语言模型的智能客服助手",
  "welcomeMessage": "您好，我是智能客服小助手，请问有什么可以帮您？",
  "llmProviderUid": "provider_123456",
  "modelUid": "model_123456",
  "systemPrompt": "你是一个专业的客服助手，请礼貌、准确地回答用户问题",
  "temperature": 0.7,
  "maxTokens": 2000,
  "isEnabled": true
}
```

## 字段说明

### 请求字段

- `name` (必需): 机器人名称
- `avatar` (可选): 机器人头像 URL
- `description` (可选): 机器人描述
- `welcomeMessage` (可选): 欢迎语
- `llmProviderUid` (必需): LLM 提供商 UID
- `modelUid` (必需): 模型 UID
- `systemPrompt` (可选): 系统提示词
- `temperature` (可选): 温度参数，0-2，默认 0.7
- `maxTokens` (可选): 最大 token 数，默认 2000
- `isEnabled` (可选): 是否启用，默认 true

### 机器人对象字段

- `uid`: 机器人唯一标识符
- `name`: 机器人名称
- `avatar`: 机器人头像 URL
- `description`: 机器人描述
- `welcomeMessage`: 欢迎语
- `llmProviderUid`: LLM 提供商 UID
- `modelUid`: 模型 UID
- `systemPrompt`: 系统提示词
- `temperature`: 温度参数
- `maxTokens`: 最大 token 数
- `isEnabled`: 是否启用
- `threadCount`: 会话数量
- `messageCount`: 消息数量
- `createdAt`: 创建时间
- `updatedAt`: 更新时间

## 使用场景

- 创建智能客服机器人
- 配置 AI 助手
- 多机器人管理
- A/B 测试不同配置

## 注意事项

- 需要先配置 LLM 提供商和模型
- 系统提示词会影响机器人的回复风格
- 温度参数控制回复的随机性
- 可以为不同场景配置多个机器人
